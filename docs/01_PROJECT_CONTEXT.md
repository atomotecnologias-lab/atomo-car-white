# Contexto do Projeto

## Nome

Primos Cockpit

---

# Cliente

Primos Automóveis

Empresa especializada na compra, venda e troca de veículos seminovos.

O projeto está sendo desenvolvido inicialmente para atender a operação da Primos Automóveis, porém toda a arquitetura deve ser pensada para futura expansão e reutilização em outras revendas.

---

# Sobre o Projeto

O Primos Cockpit é uma plataforma operacional para revendas de veículos.

O objetivo não é criar apenas um website.

O objetivo é criar um ambiente único onde o lojista consiga:

* gerenciar estoque;
* cadastrar veículos;
* publicar veículos;
* acompanhar leads;
* analisar resultados;
* automatizar tarefas operacionais;
* preparar futuras automações de marketing e atendimento.

O site público é apenas uma das interfaces do sistema.

O verdadeiro produto é a operação por trás da publicação e gestão dos veículos.

---

# Problema Atual do Mercado

O processo de cadastro e publicação de veículos normalmente exige:

* repetição de informações;
* excesso de trabalho manual;
* múltiplos sistemas;
* atualização constante de anúncios;
* acompanhamento descentralizado de leads;
* falta de indicadores claros para tomada de decisão.

Além disso, muitos lojistas dependem de processos operacionais que consomem tempo e reduzem a produtividade da equipe.

---

# Proposta de Valor

O Primos Cockpit deve reduzir esforço operacional e aumentar velocidade.

O sistema deve permitir que um veículo seja cadastrado de forma rápida e intuitiva e fique disponível para exibição no estoque online sem processos complexos.

No futuro, a plataforma deverá auxiliar o lojista através de inteligência artificial e automações operacionais.

---

# Objetivo do MVP

A primeira versão do projeto tem como objetivo demonstrar uma operação real funcionando.

O fluxo mínimo necessário é:

1. Acessar o painel administrativo.
2. Cadastrar um veículo.
3. Fazer upload das fotos.
4. Salvar o veículo no banco de dados.
5. Atualizar automaticamente o estoque público.
6. Exibir o veículo no site.
7. Visualizar o veículo em uma página individual.

Se esse fluxo funcionar de ponta a ponta, o MVP estará validado para demonstração comercial.

# Princípios do Produto

## Estoque é o principal ativo

Toda decisão de produto deve considerar que o estoque é o centro da operação.

O painel deve destacar informações relacionadas aos veículos antes de qualquer outra métrica secundária.

---

## Menos digitação

O sistema deve reduzir ao máximo atividades repetitivas.

Sempre que possível:

* sugerir;
* automatizar;
* preencher;
* simplificar.

---

## Experiência Premium

A interface deve transmitir:

* confiança;
* organização;
* tecnologia;
* profissionalismo.

O produto deve parecer uma ferramenta moderna e de alto valor agregado.

---

## Preparação para IA

Mesmo que algumas funcionalidades ainda não estejam implementadas, a arquitetura deve ser preparada para:

* geração automática de descrições;
* classificação de imagens;
* geração de conteúdo;
* análise comercial;
* recomendações operacionais.

---

# Visão de Longo Prazo

A visão futura é transformar o Primos Cockpit em uma plataforma completa para operação de revendas.

A plataforma deverá integrar:

* estoque;
* marketing;
* leads;
* atendimento;
* indicadores;
* automações;
* inteligência artificial.

O objetivo é centralizar toda a operação comercial em um único ambiente.

---

Substituir dados mockados por dados reais utilizando:

* banco de dados;

* armazenamento de imagens;

---

# Definição de Sucesso

O projeto será considerado bem-sucedido quando um lojista conseguir:

* cadastrar um veículo em poucos minutos;
* publicar o veículo;
* visualizar imediatamente no site;
* perceber ganho operacional em relação ao processo tradicional.

O sistema deve demonstrar simplicidade operacional com aparência de software premium.
